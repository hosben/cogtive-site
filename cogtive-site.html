What exactly does Cogtive do? +

Cogtive is an IoT and AI-based MES+MOM platform for optimizing industrial processes. It collects production data in real time, processes it, and delivers a clear digital view of the factory floor for managers, PPC teams, and shop floor operators. AI Vision and TÆLOR add computer vision and decision intelligence on top of that foundation.

How long does implementation take? +

Cogtive is plug-and-play. Once you upload your base configuration (equipment, products, cycle times), data starts flowing immediately. Reliable, analyzable data is typically available within 30 days, depending on team dedication. Our implementation guide is built on best practices from 300+ deployments.

Does it work on any type of equipment? +

Yes. Cogtive supports discrete and continuous production equipment via sensors, batch equipment (tanks, mixers, reactors) where duration or recipe is measurable, and manual lines. Data collection can run without any hardware investment using existing tablets.

Can it integrate with our existing ERP or BI tools? +

Yes. Cogtive exposes an open public REST API, allowing integration with any ERP or BI system with minimal effort. Our team supports integration planning as part of standard onboarding.

We already have an OEE solution. Does Cogtive add value? +

Significantly. Most OEE tools focus on individual machines. Cogtive provides a cross-equipment view of production flow — enabling bottleneck identification and system-wide optimization. TÆLOR, Smart Setup, Holding Time management, and Flow View go far beyond OEE.

Is it suitable for small factories with few machines? +

Absolutely. Cogtive pricing scales with the number of contracted equipment. Small operations access the same platform used by enterprise manufacturers — contact us for a tailored quote.

How does support work? +

Support runs via email, chat, WhatsApp, and phone, Monday to Friday from 8:00 am to 6:00 pm GMT-3. In addition to live support, we provide training, documentation, and self-service resources to maximize platform value from day one.

Calculation models

Four types of OEE.
One platform.

Choose which events impact availability. Switch between views on any dashboard without exporting to spreadsheets.

Real

OEE Real

Includes all recorded events — planned stops, unplanned downtime, changeovers. The most comprehensive view of total equipment efficiency.

Use case: global benchmark against industry standards.
Effective

OEE Effective

Excludes non-scheduled shifts and planned non-production periods. Shows efficiency only during scheduled operating windows.

Use case: shift performance review and planning.
Optimized

OEE Optimized

Removes specific losses considered acceptable by the operation. Focuses attention on the gaps that are actually actionable.

Use case: continuous improvement targeting.
TEEP

TEEP

Total Effective Equipment Performance — measures against full 24h availability. The theoretical maximum extraction from each asset.

Use case: capacity expansion and CapEx decisions.
Features & microfunctions

Every detail of your
downtime, tracked.

The OEE module handles the full complexity of industrial operations — from batch processes to QR-code-validated maintenance events.

Maintenance event grouping

When corrective maintenance occurs in sequence within a short interval, the system automatically merges them into a single downtime event. Rules are user-configurable.

If two corrective events occur within 20 minutes, Cogtive counts them as one — keeping MTBF and MTTR accurate and free of distortion.

Custom indicators

Create company-specific metrics calculated from event durations or counts. Results appear directly in OEE reports alongside standard indicators.

Example: average daily meal-break events per shift — automatically calculated and tracked.

Configurable maintenance targets

Select which event types feed into MTBF, MTTR, MTTP, MTBP, and MWT calculations. Color-coded dashboards show performance vs. target instantly.

Include or exclude adjustment events from failure metrics — aligned with your maintenance policy.

QR code event validation

Critical events can require QR code authorization before being confirmed. Only approved personnel can validate the record.

Adds a compliance layer for sensitive procedures, ensuring the right person signs off before the event is logged.

Batch process OEE

Calculates OEE based on cycle time rather than unit count — designed for tanks, reactors, mixers common in pharma, chemical, and food industries.

No quantity input required. Efficiency is measured by how long each batch takes relative to the standard cycle.

Custom event fields

Add up to three extra data fields per event — numeric, text, or list — with optional mandatory fill. Turns Pareto analysis from generic to granular.

For a corrective maintenance event: log the work order number, intervention location, and a detail note — all inside the same record.

Automated critical alerts

Notify the right people the moment a critical event occurs or exceeds a defined threshold. Channels: email, Teams, Slack, or SMS.

Production manager gets an instant alert when a machine exceeds standard setup time or when a priority run begins.

Route and speed validity periods

When process speeds improve, define new route standards with validity start dates. Historical data keeps its original standard — no retroactive distortion.

Jan–May: 800 units/hour. June onwards: 950 units/hour. Both periods calculate OEE correctly against their own baseline.

Workforce tracking

Log operator headcount per shift directly from the tablet with one tap. Correlate production efficiency with labor levels across any time window.

Identify if output drops correlate with reduced staffing — or validate that a lean team maintains target performance.

Smart setup management

Cogtive groups all setup-related intervals as a single continuous event, even if interrupted by breaks or shift changes.

A setup interrupted by a meal break is counted as one uninterrupted setup event — no fragmentation in the metrics.

Equipment grouping (avg OEE)

Combine multiple machines into a group and view weighted average OEE, consolidated maintenance targets, and group-level setup performance.

See the OEE of an entire production line as a single entity — useful for shift handovers and plant manager reviews.

Record correction & audit trail

Authorized managers can edit or delete event records. The system flags inconsistencies and maintains a full audit history of every change.

Identify process deviations — wrong route, invalid lot, timing conflicts — and correct them before they skew weekly reports.

What it does

See everything.
Act immediately.

FlowView replaces shift reports and status calls with a live map of your entire operation. If something is wrong, you see it before it becomes a problem.

01

Real-time map

Every machine, every queue, every batch — displayed in the exact sequence of your production routes. Updated continuously, no refresh needed.

02

Color-coded alerts

Orders turn yellow when approaching the time limit, red when exceeded, black when holding time has expired. Risk is visible at a glance.

03

Direct intervention

Move orders between machines, resequence queues, mark batches as priority, add comments — all directly from the FlowView screen.

Features & microfunctions

Every batch. Every
machine. Real time.

FlowView is built for PPC teams, supervisors, and operational excellence managers who need to act — not just observe.

Production flow map

Displays the full plant layout with equipment columns. Each column shows the active order at the bottom and the waiting queue above — instantly revealing which machines are idle, in setup, or running.

One screen replaces the need to walk the floor or call each supervisor for a status update.

Batch detail view

Select any order to see its full trajectory: time spent at each step, processing vs. waiting splits, and color-coded deviation from the planned route standard.

Yellow or red segments on a batch timeline immediately identify which step caused the most delay.

Priority flagging (Favorites)

Mark critical orders with a star. Favorited batches are visually highlighted and trigger automatic notifications as they progress through each production stage.

A director can be automatically notified by email whenever a strategic lot completes a stage — without checking the system manually.

Queue resequencing (FIFO)

View the complete queue for any machine and reorder it on the spot. Promote a priority lot to the front, or reorganize based on material availability — changes take effect immediately.

PCP promotes Lot B-311 to position 1 in the Coating queue without leaving the FlowView screen.

Move order to another machine

When a machine fails or capacity needs rebalancing, move an order to any compatible alternative machine configured in the route. The flow updates instantly.

Reactor R-02 goes down. PCP moves three queued batches to R-05 in seconds — no reprinting of orders needed.

Order comments

Attach timestamped notes to any order directly in FlowView. Comments are visible to all users — no need for separate messaging tools for order-level communication.

"Awaiting QC release" or "Material shortage delaying this lot" — pinned to the batch card for the whole team to see.

Flow inconsistency detection

Any order movement that deviates from the planned route — skipping a step, entering the wrong machine, starting before the previous stage is closed — is automatically flagged.

An operator starts production on the wrong equipment. FlowView flags the inconsistency in real time before the batch advances further.

Lead time & holding time alerts

Orders change color as time thresholds are approached or exceeded: yellow (at risk), red (overdue), black (holding time expired). The primary bottleneck is labeled G1.

In pharma, a batch that exceeds inter-stage holding time turns black — triggering immediate intervention before the lot must be discarded.

Interactive legend

A built-in visual legend explains every icon, color, and symbol on screen. New users can interpret the full FlowView without prior training.

Wrench icon = maintenance in progress. Clock = setup. Star = priority lot. Fully self-documented.

5 customizable filters

Toggle auto-refresh, show/hide lead time markers, display cycle time details, add a secondary reference for each lot, or show progress bars per order. Each user configures the view for their role.

PPC shows lead time markers; maintenance hides them and focuses on equipment status. Same screen, personalized per function.

WIP and lead time header

The top of every FlowView screen shows total active orders (WIP) and average lead time of recently completed lots — the two numbers that tell you if your flow is healthy.

WIP jumped from 18 to 31 orders overnight — FlowView shows it before the morning meeting even starts.

Bottleneck identification (G1)

The system automatically identifies and labels the primary flow constraint — the machine with the longest queue relative to its capacity — as G1 directly on the map.

No analysis required. The bottleneck is labeled on screen. The team knows exactly where to focus first.

What it does

Production and demand,
synchronized.

Pace Management is the S&OP layer on top of your shop floor data. It answers the questions your production team asks every morning without building a single spreadsheet.

Will we meet this month demand?

The realized bar shows exactly where you stand vs. plan. The WIP layer shows what is still in progress. The gap is visible at a glance, updated in real time.

Which SKUs are at risk of stockout?

Coverage status (Critical, Low, Optimal, Full, Excess) is calculated automatically per SKU. Critical means act now. Low means watch closely.

What daily pace do we need to recover?

The new pace calculation factors in what has been produced, what is in WIP, and the days remaining in the period — giving a concrete daily target.

Features & microfunctions

Every SKU. Every day.
Full visibility.

Pace Management consolidates demand planning, WIP tracking, inventory coverage, and production fulfillment into a single real-time interface.

Demand registration & import

Import planned demand via a standardized Excel file per SKU identifying the final production stage and daily quantities. The system validates every SKU against existing routes before accepting.

Any SKU without a configured production route is flagged before import.

Period performance overview

The main screen shows total planned vs. actual for the selected period with quick filters. Updated automatically as production data flows in from the shop floor.

At 8am the production manager sees the accumulated gap without opening any spreadsheet.

Entry rate (order launch pace)

Chart showing how many production orders have been released over time compared to the planned cadence. Reveals if order launching is keeping up with demand.

If orders are released faster than completed lots, WIP is accumulating. If slower, a supply gap is forming.

Output rate (completed production pace)

Mirror chart of the entry rate — showing actual completed lots vs. the planned demand curve. The gap is the production deficit or surplus at any point in time.

Entry rate vs. output rate shows immediately if the bottleneck is in order launching or in execution.

Inventory coverage & stock status

For each SKU, calculates days of coverage based on average monthly demand and current stock. Classifies automatically into five tiers from Critical to Excess.

Product B shows Critical — 41% coverage. The team knows before the week starts that an expedite is needed.

% Realized and WIP progress bars

Dual progress bar per SKU: percentage produced (Realized) and in process (WIP). Green at or above 100%, yellow 80 to 99%, red below 80%.

Product A at 72% realized plus 18% WIP equals 90% total. New pace is calculated automatically.

WIP completion forecast

Calculates when all open lots will be completed based on the average lead time of the last 100 finished lots and the remaining steps of each open order.

The forecast shows Product B lots will complete 4 days after month-end. Intervention is still possible.

Flow position heat map

Color intensity chart showing where in the production flow the open lots of each SKU are concentrated. Dark blue means high concentration.

All Product C lots are in the final stage. All Product D lots are in stage 1. Same WIP, very different risk profile.

New pace calculation

For each SKU behind target, calculates the daily production rate needed from today to still meet the period goal. Factors in realized and WIP quantities.

Product B needs 847 units per day for the remaining 12 days. The team has a concrete number to plan overtime.

Lead time and quantity consolidation per SKU

Each product row shows: planned quantity, WIP, realized, remaining balance, average lead time of recent lots, and the new pace required — all in one line.

A single row gives the planning team a 360 view of each product without switching screens.

Features & microfunctions

Every changeover.
Measured and improved.

Smart Setup combines step-by-step operator guidance with automatic time capture, ramp-up tracking, and historical validity periods.

Step-by-step changeover guide

Each product-equipment combination can have a defined procedure with numbered steps shown on the operator tablet during setup. Consistent regardless of who performs it.

A new operator follows the same 11-step procedure as a 10-year veteran. Setup time variability drops immediately.

Automatic setup time capture

Setup time is measured from the moment the operator registers the setup event to the start of the next production run. No stopwatches, no manual forms, no estimation.

The system captures 22 minutes. The standard is 18. The gap is visible in the report — automatically, every time.

Actual vs. standard time comparison

Every completed setup is compared against the configured standard time. Deviations are flagged, trended, and included in OEE impact calculations.

Product A setups on Reactor R-04 average 24 minutes vs. a 20-minute standard. The report shows the trend over the last 30 days.

Setup interruption handling

If a setup is interrupted by a shift change, meal break, or unplanned event, Cogtive groups all the intervals into a single continuous setup event. No fragmentation in the metrics.

Operator starts setup at 11:40, pauses for lunch, resumes at 13:00, finishes at 13:22. Total setup time: 1h 42min — one record, not three.

Ramp-up tracking

After setup completes, the ramp-up period — from first unit to stable production speed — is tracked separately. The Ramp-up Report shows availability and performance during this critical window.

Line 3 takes an average 34 minutes to reach nominal speed after each changeover. Smart Setup makes this visible — and improvable.

Setup standard validity periods

When a process improvement reduces the standard setup time, the new standard is registered with a start date. Historical data is preserved against the old standard — no retroactive distortion.

Jan to Apr: 25-minute standard. May onwards: 20-minute standard after a method improvement. Both periods analyze correctly against their own baseline.

Proportional calculation on partial periods

When a report is filtered by a partial period, Cogtive adjusts setup quantities and averages proportionally — ensuring metrics remain accurate for any time window.

A manager filters the last 4 hours. Setup count and average time are adjusted to that window — not the full shift total.

Setup overview for resource planning

The setup overview dashboard shows all machines currently in setup, their elapsed time vs. standard, and estimated completion — a real-time picture of changeover load across the plant.

At 14:00 the plant manager sees 4 machines in setup simultaneously. Two are on track. One is overdue by 12 minutes. One is in ramp-up.

Integration with OEE

Every setup event feeds directly into OEE calculations. Setup time beyond the standard reduces Availability. Ramp-up inefficiency reduces Performance. The full cost of each changeover is reflected in OEE.

A 30-minute setup on a machine with a 20-minute standard contributes 10 minutes of avoidable availability loss to the daily OEE calculation.
